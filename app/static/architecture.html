PAYBACK Lightweight Assistant — Architecture

How three disparate partner datasets are indexed and queried simultaneously: each partner keeps its own taxonomy, ingestion normalizes everything into one shared schema, and a query hits a single unified in-memory index (serving path) while the same catalogs live in three separate BigQuery tables queried at once via VECTOR_SEARCH … UNION ALL (warehouse path). The API runs on Cloud Run; queries the classifier cannot ground in the catalog escalate to Gemini 2.5 Flash-Lite served on Vertex AI (service-account auth, no keys). ← back to the demo UI
